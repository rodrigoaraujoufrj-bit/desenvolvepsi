# Hero - Layout

## Largura e grid

A Hero utiliza largura maxima ampla de 1200px, alinhada ao container wide do
projeto. No desktop, o grid deve ter duas colunas equilibradas, com leve
prioridade para a coluna textual.

- Coluna esquerda: aproximadamente 52% da largura disponivel.
- Coluna direita: aproximadamente 48% da largura disponivel.
- Gap desktop: entre 48px e 64px.
- Gutter lateral: herdado do container global.

## Alinhamentos

O conteudo textual deve ficar alinhado a esquerda no desktop. O bloco visual
deve ficar centralizado na propria coluna, com altura suficiente para criar
ponto focal sem competir com a headline.

## Espacamentos

A secao deve ocupar praticamente a primeira dobra, considerando a navbar fixa.
O ritmo vertical deve manter respiro sem gerar vazio excessivo.

- Topo: compensar a navbar sem empurrar a Hero para baixo demais.
- Headline e subheadline: proximidade clara de grupo.
- CTAs: posicionados logo apos a subheadline.
- Indicadores: abaixo dos CTAs, como reforco de confianca.

## Proporcoes

A headline e o maior elemento textual da dobra. O bloco visual deve ter escala
comparavel, mas nao deve ser mais dominante que a mensagem principal. A foto
profissional deve ocupar a maior parte do painel visual quando estiver
disponivel.

## Hierarquia visual

1. Headline.
2. CTA principal.
3. Foto profissional.
4. Subheadline.
5. CTA secundario.
6. Indicadores de confianca.
7. Elementos decorativos.

## Ordem dos elementos

A leitura deve seguir de cima para baixo na coluna textual: contexto, mensagem,
explicacao, acao e confianca. A coluna visual deve sustentar emocionalmente a
mensagem, sem adicionar informacoes indispensaveis.

## Desktop

Manter duas colunas, altura minima proxima ao viewport util e composicao
visual com formas organicas suaves. A area direita deve estar preparada para
foto real, com camada de profundidade sutil.

## Tablet

Empilhar ou reduzir para duas colunas mais compactas conforme o espaco
disponivel. A headline deve continuar forte, mas sem quebrar em linhas
excessivas. CTAs podem permanecer lado a lado se houver largura confortavel.

## Mobile

Empilhar conteudo. A mensagem vem antes da imagem. CTAs devem ocupar largura
facil de tocar. Indicadores devem virar lista compacta, com icones pequenos e
boa distancia entre itens. A area visual deve ter altura menor para nao
empurrar o restante da pagina.
