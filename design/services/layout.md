# Services - Layout

## Largura e grid

Usar container wide de ate 1200px. No desktop, a secao pode dividir conteudo
em uma coluna de titulo e uma area de cards. A area de cards deve usar grid de
duas colunas quando houver espaco suficiente.

## Alinhamentos

O titulo fica alinhado a esquerda no desktop para manter continuidade com a
jornada. Os cards devem manter alinhamento interno consistente: icone, titulo
e descricao.

## Espacamentos

Aplicar espacamento vertical generoso entre secoes, seguindo o Design System.
Entre cards, usar gap compacto o bastante para comparacao rapida, sem parecer
denso demais.

## Proporcoes

Cada card deve ter peso visual equivalente. Nenhuma demanda deve parecer mais
importante sem decisao explicita de produto.

## Responsividade

- Desktop: titulo lateral e grid de cards em duas colunas.
- Notebook: manter grid de duas colunas se a leitura continuar confortavel.
- Tablet: titulo acima ou coluna reduzida, cards em duas colunas.
- Mobile: todos os cards em uma coluna.

## Hierarquia visual

1. Titulo da secao.
2. Descricao orientadora.
3. Titulos dos cards.
4. Descricoes.
5. Icones.

## Ordem dos elementos

A secao deve partir da promessa ampla para casos especificos. Primeiro
explicar o tipo de acolhimento, depois listar demandas reconheciveis.
